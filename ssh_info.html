Introduction to SSH (Secure Shell)

Secure Remote Login

When you type commands in your local terminal, the characters you input are sent from your keyboard to the shell, which displays them on the screen. If you type a command, the shell executes it and shows the output. To execute commands on another machine, such as a server managing a database, you need to log in to that remote machine. This process is known as a remote login. For a successful remote login, the remote computer must be running a remote login server, while you will use a client program to connect to this server. The client sends your login credentials to the server, which, if authenticated, launches a shell on the remote computer for you. The SSH (Secure Shell) protocol was designed to secure remote logins by encrypting the network traffic, thus protecting against eavesdropping, connection hijacking, and other attacks. SSH uses sophisticated encryption to ensure that communications between the client and server are secure.

Key Concepts

  • SSH: A secure alternative to remote login and file transfer.
  • SSH Daemon (sshd): The server-side program that accepts connections from SSH clients.
  • SSH Client: The program used to connect to the remote SSH server.
  • SSH Keys: Generated in pairs (public and private). The public key can be shared, while the private key remains on your machine.

Using SSH

To connect to a remote computer, run the following syntax in your command line:

ssh username@remoteComputerAddress

To verify which system your terminal is connected to, run: hostname. After logging in, you can use the remote shell as if it were your local shell.

To exit the remote session, type exit or press Control-D.

Secure Copy Protocol (SCP)

SCP (Secure Copy Protocol) is used to copy files to or from a remote machine over the SSH connection. It ensures that file transfers are secure.

Copying Files

To securely copy of file from your local computer onto a remote server, use:

scp localFile.txt username@remoteComputerAddress:/path/to/remote/directory/

To securely copy a file from a remote server to your local computer, use:

scp username@remoteComputerAddress:/path/to/remoteFile.txt /local/directory/

Copying Directories

To securely copy a directory recursively (i.e., copy everything inside too), use the -r flag:

scp -r username@remoteComputerAddress:/path/to/remoteDirectory/ /local/directory/

Important Note

The colon : is used to differentiate between local and remote paths. For example:

scp file.txt user@remotehost:/remote/path/

Here, file.txt is copied to /remote/path/ on remotehost. Without the colon, SCP will attempt to create a file named user@remotehost on your local machine.

High Performance Computing (HPC)

HPC systems are used for computationally intensive tasks and often involve connecting to large computing systems remotely. Interacting with these systems is typically done via a command-line interface.

Modules in HPC

Modules are used to manage software environments in HPC systems. They allow users to switch between different software versions and manage dependencies.

Module Commands

  • List Available Modules: module avail
  • Retrieve Specific Module: module spider <software-name>
  • Load a Module: module load <software-name>
  • List Loaded Modules: module list
  • Swap Modules: module swap <old-module> <new-module>
  • Unload a Module: module unload <software-name>
  • Purge All Modules: module purge
  • Reset Modules: module reset
  • Help with Modules: module help

For more information on module commands, refer to this documentation.
